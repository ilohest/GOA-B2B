Je démarre un projet de POC pour interconnecter une application web de prise de commande en Vue 3 (Vite + TypeScript) avec l'API de notre ERP métier Easybeer.À la racine du projet, tu trouveras deux fichiers essentiels :easybeer-api-docs.json : La spécification OpenAPI/Swagger complète de l'API cible.  poc-development-plan.md : Le plan de développement du POC.Lis attentivement ces deux fichiers. D'après la documentation, l'API utilise une authentification standard Basic Auth (identifiant/mot de passe de l'utilisateur) et la base URL est api.easybeer.fr.  Pour cette première étape, aide-moi à mettre en place la Phase 1 et la Phase 2 du plan :Propose-moi la configuration du fichier .env.local pour y stocker en toute sécurité mes identifiants Easybeer de test.Configure le proxy de développement dans mon fichier vite.config.ts (ou écris-moi un mini script Node.js si tu penses que c'est plus robuste pour gérer le Basic Auth et éviter les problèmes de CORS). Ce proxy doit écouter sur /api, intercepter les requêtes du front, générer la chaîne Base64 du Basic Auth, et la pousser vers [https://api.easybeer.fr](https://api.easybeer.fr).  Écris un service d'API (easybeerService.ts) en TypeScript utilisant fetch ou axios qui appelle l'endpoint POST /parametres/client/liste en consultant le modèle ModeleClientFiltre du JSON pour savoir quels paramètres de pagination envoyer par défaut.  Crée un composant Vue simple (ClientsTest.vue) qui appelle ce service au montage (onMounted), récupère la liste des clients pros et les affiche dans un simple tableau HTML propre.Ne commence pas à coder la partie commande pour le moment, concentrons-nous sur la configuration du proxy et l'affichage des clients.
